Awaiting User Clearance

Dennis Grzegorzewski, 24 November 2014 (created 3 November 2014).

UNFINISHED.

Description:

This status queue holds requests placed by users who are listed in the Users To Clear queue in the System section, on the Main page in ILLiad, because they have not yet been cleared. Requests are automatically routed to this status queue from the Awaiting Request Processing, Awaiting User Expiration Updating, or Awaiting Citation Completion status queues when the user has not been cleared.

To process:

DO NOT ROUTE REQUESTS FROM THIS QUEUE

  1. To process simply open and close each request.
    1. If the user has been cleared, requests will automatically be routed...
      • to the Awaiting User Expiration Updating status queue if user has been cleared, but the user's expiration date has passed,
      • or to the Awaiting Citation Completion status queue if user has been cleared, and the user's expiration date has not passed.
  3. Requests remain in this queue until the patron is cleared or disavowed in the Clear User process

More information:

When requests are routed to this queue they are automatically routed...

  • to the Awaiting User Expiration Updating status queue if user has been cleared, but the user's expiration date has passed,
  • or to the Awaiting Citation Completion status queue if user has been cleared, and the user's expiration date has not passed.

This information has been updated for ILLiad 8.5.2.0.
