Awaiting User Expiration Updating

Dennis Grzegorzewski, 24 November 2014 (created 3 November 2014).

UNFINISHED.

Description:

This status queue holds requests placed by cleared users whose expiration date has been passed. At this step users are updated using Voyager or Banner. Requests are automatically routed to this queue from the Awaiting Request Processing, Awaiting User Clearance, or Awaiting Citation Completion queues when the user is cleared, but their expiration date has been passed.

To process:

Student Worker processing using Voyager.

  1. Open the users record in ILLiad.
    1. Click on the View icon in the User section of the Borrowing Process ribbon in the Borrowing Request window.
      1. This opens the User window.
    2. Copy the users barcode from the ID field in the Details section of the User Information tab.
  2. Search for the patron in Voyager Circulation.
    1. Open Voyager Circulation.
    2. Click on the Patron icon, or right click on the Patron Record window and click on Search for Patron... option in the drop-down menu, if it is already open.
      1. This opens the Patron Search window.
    3. Click on the Barcode radio button.
      1. This will select searching by barcode.
    4. Paste the users ID in the Barcode: input field.
    5. Click the Find button.
      1. This will search for the patron and opens the Patron Record window, if it is found.
      2. If the patron is not found, try searching by name, or just close the request and leave it in the queue.
    6. Click on the History tab, and examine the date in the Expires: field.
  3. Update their expiration in ILLiad as indicated.
    1. If the date in the Expires: field on the History tab in the Patron Record window in Voyager is later than the date in the Expiration Date field in the ILLiad User record window, then copy and paste the Expires: date into the Expiration Date field.
    2. Save the changes if necessary, by clicking on the Save icon in the upper left corner.
  4. Close the User record window.
  5. Close the Borrowing Request window.
  6. If the user is no longer expired, the request will be routed to the Awaiting Citation Completion queue, otherwise it is left in the current queue.

Staff processing using Banner.

Student workers can check user expiration dates in Voyager, but they are unable to check in Banner. When a users registration is expired in Voyager, they still may be registered for classes, but this information may not yet be updated in Voyager. A check in Banner is necessary to definitively check for expiration.

  1. Open the users record in ILLiad.
    1. Click on the View icon in the User section of the Borrowing Process ribbon in the Borrowing Request window.
      1. This opens the User window.
    2. Copy the users barcode from the ID field in the Details section of the User Information tab.
  2. Open the Registration Query [SFAREGQ] screen in Banner to determine if a student is currently registered for classes.
    1. Select the term to display in the Term: input field.
    2. Paste the users ID in the ID: input field.
    3. Click on the Next Block icon.
      1. This will display classes registered in for the term selected.
    4. If the student is registered for credit hours during the term in question, update his Expiration Date in ILLiad to the end of the term.
      1. Click on the Save icon, to save the changes.
      2. Close the User record window.
      3. Close the Borrowing Request window.
      4. The request will be routed to the Awaiting Citation Completion queue.
  3. If the student is not registered for classes, open the General Student [SGASTDN] screen in Banner to determine if a student's status is active, inactive or graduated, or if they are a doctoral student.
    1. Paste the users ID in the ID: input field.
    2. Click on the Next Block icon
    3. If the student has any of the following statuses they are inactive. Block their account in ILLiad, and email route their request to the No Privileges - Do Not Process status queue.
      1. For the statuses of Academic Dismissal, No Affidavit of Support, Admit Incomplete, Application Withdrawn, Disciplinary Expulsion, Student withdraw, or Inactive
        1. Type INACTIVE in the Number field of the ILLiad user record.
        2. Click on the Blocked icon.
        3. Click on the Save icon, to save the changes.
        4. Close the User record window.
        5. In the Borrowing Request window,
          1. Double click the Note field to create a new note.
          2. Enter the date, one week from today, in the following format - DELETE MM/DD/YYYY.
          3. Click the Add button to save.
          4. Copy and paste this text into the Special Instructions field.
          5. Click on the Save icon, to save the changes.
          6. Click on the Send E-Mail icon
          7. Click on the Expired Privileges dropdown menu item
            1. This opens the Edit Email Form
          8. Click on the Send button.
            1. This sends a notification email, closes the Borrowing Request window, and routes the request to the No Privileges - Do Not Process status queue.
      2. For the status of Inactive due to Graduation
        1. Type GRADUATED in the Number field of the ILLiad user record.
        2. Change the Illiad user Status to Alumni.
        3. Click on the Blocked icon.
        4. Click on the Save icon, to save the changes.
        5. Close the User record window.
        6. In the Borrowing Request window,
          1. Double click the Note field to create a new note.
          2. Enter the date, one week from today, in the following format - DELETE MM/DD/YYYY.
          3. Click the Add button to save.
          4. Copy and paste this text into the Special Instructions field.
          5. Click on the Save icon, to save the changes.
          6. Click on the Send E-Mail icon
          7. Click on the No Privileges (Alumni) dropdown menu item
            1. This opens the Edit Email Form
          8. Click on the Send button.
            1. This sends a notification email, closes the Borrowing Request window, and routes the request to the No Privileges - Do Not Process status queue.
      3. For the status of Leave of Absence
        1. Type LEAVE OF ABSENCE in the Number field of the ILLiad user record.
        2. Click on the Blocked icon.
        3. Click on the Save icon, to save the changes.
        4. Close the User record window.
        5. In the Borrowing Request window,
          1. Double click the Note field to create a new note.
          2. Enter the date, one week from today, in the following format - DELETE MM/DD/YYYY.
          3. Click the Add button to save.
          4. Copy and paste this text into the Special Instructions field.
          5. Click on the Save icon, to save the changes.
          6. Click on the Send E-Mail icon
          7. Click on the Expired Privileges dropdown menu item
            1. This opens the Edit Email Form
          8. Click on the Send button.
            1. This sends a notification email, closes the Borrowing Request window, and routes the request to the No Privileges - Do Not Process status queue.
    4. If the student in not inactive and has the Class: Graduate Doctoral, update their expiration to the end of the current term or intersession.
      1. Type ACTIVE DOCTORAL in the Number field of the ILLiad user record.
      2. Click on the Save icon, to save the changes.
      3. Close the User record window.
      4. Close the Borrowing Request window.
      5. The request will be routed to the Awaiting Citation Completion queue.

More information:

When requests are routed to this queue they are automatically routed...

  • to the Awaiting User Clearance status queue if the user has not been cleared,
  • or to the Awaiting Citation Completion status queue if user has been cleared, and the user's expiration date has not passed.

This information has been updated for ILLiad 8.5.2.0.
